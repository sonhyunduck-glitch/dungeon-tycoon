# 던전 & 상점 타이쿤 — 텍스트 웹게임 기획서

> 유저가 던전을 사냥해 아이템을 습득하고, 습득한 아이템을 직접 착용하거나
> 자신이 운영하는 상점에서 판매하는 **헌터 + 상점 타이쿤** 융합 텍스트 게임

- **작성일**: 2026-06-14
- **플랫폼**: PC / 모바일 웹 (반응형)
- **기술 스택**: HTML + CSS + Vanilla JavaScript (프레임워크 無, 로컬 저장 기반)

---

## 1. 게임 개요

### 1.1 한 줄 소개
낮에는 던전을 사냥하는 **헌터**, 밤에는 가게를 운영하는 **상점주**가 되는 1인 텍스트 경영 RPG.

### 1.2 핵심 재미 (Core Loop)
```
던전 탐험 → 전투 → 전리품 획득 → 아이템 분류
        ↘ 착용(강해짐) ──→ 더 깊은 던전 도전
        ↘ 진열/판매 ──→ 골드 획득 ──→ 상점 확장 / 장비 강화
```
"강해져서 더 좋은 걸 줍고 → 더 비싸게 팔고 → 가게를 키우고 → 더 강해진다"는 양방향 성장 루프가 핵심.

### 1.3 차별점
- NPC 상인이 아니라 **유저 본인이 상점을 운영**하는 타이쿤형 구조
- 같은 아이템도 **착용할지 / 팔지** 매번 선택하는 자원 분배의 긴장감
- 텍스트 기반이라 가볍고 어디서나 즉시 플레이 가능

## 3. 핵심 시스템 설계

### 3.1 던전 시스템

#### 스테이지 구성 (노드형)
하나의 **스테이지 = 여러 개의 노드(칸)가 일렬로 연결**된 형태. 마지막 노드는 항상 보스.

```
□ ─ □ ─ □ ─ ■
1   2   3   보스
```

- `□` **일반 노드**: 진입하면 이벤트(전투/보물/함정/휴식/정예) 발생 → 처리해야 다음 노드 해금
- `■` **보스 노드**: 스테이지 최종 관문, 처치 시 **스테이지 클리어 + 확정 희귀 드랍**
- **선형 진행**: 왼쪽→오른쪽으로 한 칸씩 전진(이전 노드로 후퇴 불가), 진행 상황을 노드 막대로 시각화
- 노드 표시 상태: `■`(클리어) `▶`(현재) `□`(미도달) `🔒`(잠김)

```
■ ─ ■ ─ ▶ ─ □ ─ ■    ← 3번째 노드 진행 중
```

#### 스테이지 / 층 구조
- **층(Floor)** 1개 = **스테이지(Stage)** 여러 개 → 마지막 스테이지 보스 처치 시 다음 층 해금
  ```
  [1층]  Stage 1-1 → 1-2 → 1-3(층 보스) → [2층 해금]
  ```
- 각 층마다 **권장 전투력**, 등장 몬스터, 드랍 테이블이 다름
- 노드 수는 스테이지가 깊어질수록 증가(예: 1-1은 3노드+보스, 1-3은 6노드+보스)
- 클리어한 스테이지는 **재도전(파밍) 가능** → 아이템/골드 반복 수급처

#### 노드 진입 시 이벤트 확률
일반 노드(`□`)에 들어가면 아래 표대로 이벤트가 롤된다.

| 이벤트 | 확률(예) | 효과 |
|--------|---------|------|
| 일반 전투 | 55% | 몬스터 처치 → 골드/아이템 |
| 보물상자 | 15% | 즉시 아이템/골드 |
| 함정 | 10% | HP 감소 |
| 휴식처 | 10% | HP 회복 |
| 정예 몬스터 | 10% | 고난도, 고확률 희귀 드랍 |

> 보스 노드(`■`)는 이벤트 롤 없이 **항상 보스 전투** 고정.

#### 데이터 구조 예시
```js
// floors.js
{
  floor: 1,
  recommendPower: 50,
  stages: [
    {
      id: "1-1",
      nodes: [
        { type: "battle" }, { type: "normal" }, { type: "normal" },
        { type: "boss", monster: "goblin_lord" }   // ■ 마지막 노드
      ],
      clearReward: { gold: 100, dropTable: "boss_1_1" }
    }
    // 1-2, 1-3 ...
  ]
}
// type: "normal"=이벤트 롤, "battle"=전투 확정, "boss"=보스
```

### 3.2 전투 시스템 (텍스트 턴제)
- 스탯: **HP / 공격력(ATK) / 방어력(DEF) / 치명타율 / 속도**
- 자동 턴제(빠른 진행) + 선택지(스킬/포션 사용) 혼합
- 데미지 공식(예): `damage = max(1, ATK * 랜덤(0.9~1.1) - DEF*0.5)`, 치명타 시 1.5배
- 사망 시: 마을로 귀환, 미정산 전리품 일부 손실 (페널티)

### 3.3 아이템 시스템
- **타입**: 무기 / 방어구(투구·갑옷·장갑·신발) / 장신구 / 소모품 / 재료
- **등급(Rarity)**: 일반 → 고급 → 희귀 → 영웅 → 전설 (색상 구분)
- **속성**: 기본 스탯 + 랜덤 옵션(접사) → 같은 이름도 가치가 다름
- 각 아이템은 `기준판매가`를 가지며 등급/옵션에 따라 가치 변동

```js
// 아이템 데이터 구조 예시
{
  id: "itm_001",
  name: "낡은 강철검",
  type: "weapon",
  rarity: "uncommon",
  stats: { atk: 12 },
  affixes: [{ stat: "crit", value: 5 }],
  basePrice: 80,
  stackable: false
}
```

### 3.4 인벤토리 & 착용
- 슬롯 제한 인벤토리(확장 가능) → 가득 차면 정리 압박 → 판매 동기 부여
- 장비 슬롯: 무기 / 투구 / 갑옷 / 장갑 / 신발 / 장신구 ×2
- 착용 시 스탯 즉시 반영, 비교 UI(현재 장비 vs 신규 아이템)

### 3.5 상점 타이쿤 시스템 ⭐ (핵심 차별 요소)
유저가 직접 운영하는 가게.

- **진열대**: 아이템을 진열 슬롯에 올리고 **판매 가격을 직접 설정**
- **손님 시스템**: 시간/방치에 따라 손님 방문 → 가격·수요에 따라 구매 결정
  - 가격이 시세보다 비싸면 안 사고, 싸면 빨리 팔리지만 손해
  - 적정가 책정이 핵심 재미
- **상점 등급/확장**: 골드로 진열 슬롯 증가, 손님 유입량 증가, 인테리어
- **방치 수익(Idle)**: 접속 안 한 동안에도 일정 시간 자동 판매 → 재접속 시 정산
- **시세 변동**: 아이템별 수요가 날마다 변동(특정 아이템 인기 상승 이벤트)

```
[판매 흐름]
던전 전리품 → 인벤토리 → 진열대에 등록 + 가격 설정
            → 손님 방문 → 가격 수용 시 판매 → 골드 획득
```

### 3.6 성장 & 경제
- **골드**: 상점 판매 / 던전 골드 드랍으로 획득
- **사용처**: 장비 강화, 상점 확장, 인벤토리 확장, 포션 구매
- **강화 시스템**: 골드 + 재료로 장비 +1, +2... (성공/실패 확률)
- 골드 싱크(소비처)를 충분히 둬 인플레이션 방지

## 4.5 비주얼 디렉션 (CSS 전용, 그래픽 에셋 無)

이미지 리소스 없이 **CSS만으로** 던전 분위기를 표현한다. 색·그라데이션·테두리·그림자·이모지·CSS 패턴/애니메이션으로 텍스처를 대신한다.

### 4.5.1 컨셉
어둡고 습한 **지하 던전** → 검정/짙은 남보라 바탕에 **횃불빛(주황 글로우)**을 포인트로.
상점은 같은 톤을 유지하되 **따뜻한 황금빛**으로 분위기 전환.

### 4.5.2 컬러 토큰 (CSS 변수)
```css
:root{
  /* 베이스 — 던전 */
  --bg-0:#0d0b14;          /* 가장 어두운 배경 */
  --bg-1:#161324;          /* 패널 */
  --bg-2:#221c33;          /* 카드/입력 */
  --line:#3a2f57;          /* 경계선(보랏빛 돌벽) */
  --text:#e8e2d4;          /* 양피지톤 본문 */
  --text-dim:#9b91b0;      /* 보조 텍스트 */

  /* 포인트 — 횃불/위험/회복 */
  --torch:#ff8a3c;         /* 횃불 주황(주 강조) */
  --torch-glow:rgba(255,138,60,.35);
  --hp:#e5484d;            /* HP/위험 */
  --heal:#4fbf6b;          /* 회복/성공 */
  --gold:#f2c14e;          /* 골드/상점 */
  --mana:#5b8dff;          /* 마나/정보 */
}
/* 상점 탭에서는 톤 전환 */
.theme-shop{ --bg-0:#140f08; --torch:var(--gold); --line:#5a4420; }
```

### 4.5.3 등급(Rarity) 색상 — 텍스트 게임 핵심 시각 단서
```css
.r-common  { color:#c8c8c8; }                 /* 일반 — 회색 */
.r-uncommon{ color:#5fd17a; }                 /* 고급 — 초록 */
.r-rare    { color:#5b8dff; }                 /* 희귀 — 파랑 */
.r-epic    { color:#b57bff; }                 /* 영웅 — 보라 */
.r-legend  { color:#ffb13c;                   /* 전설 — 주황 */
             text-shadow:0 0 8px var(--torch-glow); }
```
- 인벤토리/드랍 로그/상점 진열에서 **이름 색 + 테두리 색**으로 등급 즉시 구분
- 전설 등급은 은은한 글로우 애니메이션으로 특별함 강조

### 4.5.4 배경 텍스처 (이미지 없이 CSS만)
```css
/* 돌벽 느낌 — 미세 노이즈 + 비네팅 */
body{
  background:
    radial-gradient(120% 90% at 50% 0%, transparent 60%, #000 100%), /* 비네팅 */
    repeating-linear-gradient(0deg, #1a1626 0 2px, #15111f 2px 4px), /* 돌결 */
    var(--bg-0);
}
/* 던전 탭 — 위쪽 횃불 빛이 스며드는 효과 */
.dungeon-view{
  background:radial-gradient(60% 40% at 50% -5%, var(--torch-glow), transparent 70%);
}
/* 던전 노드 막대 □─□─□─■ */
.node{ border:2px solid var(--line); border-radius:6px; background:var(--bg-2); }
.node.clear{ border-color:var(--heal); }
.node.current{ border-color:var(--torch); box-shadow:0 0 12px var(--torch-glow); }
.node.boss{ border-color:var(--hp); }
.node.locked{ opacity:.4; }
```

### 4.5.5 CSS 연출(애니메이션)
- **횃불 깜빡임**: 포인트 글로우에 `@keyframes flicker`로 미세한 밝기 변화
- **데미지 텍스트**: 숫자가 위로 떠오르며 사라지는 `float-up`
- **전설 드랍**: 화면 가장자리 골드 글로우 펄스 1회
- **HP바**: `transition:width .3s`로 부드럽게 감소, 낮으면 붉게 점멸
```css
@keyframes flicker{ 0%,100%{opacity:1} 50%{opacity:.85} }
.torch-glow{ animation:flicker 2.4s infinite; }
```

### 4.5.6 타이포 & 톤
- 본문: 시스템 폰트 스택, 로그/스탯은 **고정폭 폰트**(`ui-monospace`)로 RPG 콘솔 느낌
- 제목/수치 강조에 `letter-spacing`과 양피지톤(`--text`) 사용
- 다크 테마 단일 운영(눈 피로↓, 던전 분위기 일관) — 추후 라이트/하이콘트라스트 옵션 가능

### 4.5.7 접근성
- 색만으로 정보 전달하지 않기 → 등급은 **색 + 텍스트 라벨(\[희귀\])** 병기
- 명도 대비 WCAG AA 충족(본문 vs 배경), 글로우/점멸은 과하지 않게

## 5. 기술 설계

### 5.1 파일 구조
```
/text
├── index.html          # 메인 진입점, 탭 컨테이너
├── /css
│   └── style.css       # 반응형 레이아웃 + 등급 색상 테마
├── /js
│   ├── main.js         # 진입점, 탭 라우팅, 게임 루프
│   ├── state.js        # 전역 게임 상태(단일 store 객체)
│   ├── dungeon.js      # 던전/이벤트 로직
│   ├── combat.js       # 전투 계산
│   ├── item.js         # 아이템 생성기(랜덤 옵션/등급 롤)
│   ├── inventory.js    # 인벤토리/착용 관리
│   ├── shop.js         # 상점/손님/방치 수익
│   ├── ui.js           # 렌더링 + DOM 갱신
│   └── save.js         # localStorage 저장/불러오기
└── /data
    ├── monsters.js     # 몬스터 테이블
    ├── items.js        # 아이템 베이스 데이터
    └── floors.js       # 층별 설정/드랍 테이블
```

### 5.2 상태 관리
- 단일 전역 `gameState` 객체 → 변경 시 `render()` 호출하는 단방향 흐름
- 의존성 없는 Vanilla JS, 모듈은 ES Module(`import/export`) 또는 전역 네임스페이스

### 5.3 저장 시스템
- `localStorage`에 JSON 직렬화 저장 (자동 저장 + 수동 저장)
- 방치 수익 계산용 **마지막 접속 시각(timestamp)** 저장 → 재접속 시 경과시간 정산
- 세이브 내보내기/가져오기(텍스트 코드) 기능 제공

### 5.4 반응형 (PC / 모바일 세로 / 모바일 가로)
공통: CSS Flexbox/Grid + `clamp()` 가변 폰트, 터치/클릭 공용 이벤트, 버튼 최소 44px(터치 타깃)

세 가지 레이아웃 모드를 **방향(orientation) 기준으로 분리**한다.

| 모드 | 감지 조건 | 레이아웃 |
|------|-----------|----------|
| PC | `min-width: 1024px` | 사이드 메뉴 + 2~3열 콘텐츠 |
| 모바일 세로 | `max-width: 1023px` & `orientation: portrait` | 1열 + 하단 탭바 |
| **모바일 가로** | `max-width: 1023px` & `orientation: landscape` | 좌측 세로 탭바 + 우측 2분할(메인/로그) |

#### 모바일 가로 전용 UI (핵심 변경)
가로에서는 세로 높이가 부족하므로 **좌→우 흐름**으로 재배치한다.

```
┌──────┬──────────────────────┬─────────────┐
│ 탭   │   메인 패널           │  로그/정보   │
│ 바   │  (던전/인벤/상점 등)  │  (전투·판매  │
│(세로)│                       │   로그 스크롤)│
└──────┴──────────────────────┴─────────────┘
```

- **하단 탭바 → 좌측 세로 탭바**로 전환 (엄지 도달 영역 고려, 아이콘 위주)
- **상단 HUD → 가로 1줄 압축**(HP/골드/층/전투력을 한 줄에 인라인 배치)
- **로그 영역을 우측 고정 열**로 분리 → 메인 조작과 동시에 결과 확인
- 던전 전투·전투 로그처럼 "조작 + 결과"가 동시에 보여야 하는 화면에서 가로가 유리
- 회전 시 `orientation` 미디어쿼리 + `matchMedia` 리스너로 즉시 레이아웃 재배치(상태 유지)

#### 구현 방식
- 레이아웃 차이는 **CSS만으로** 처리(미디어쿼리), JS는 동일 DOM 유지 → 회전해도 상태/스크롤 보존
- `<meta name="viewport">`로 확대 방지, `dvh` 단위로 가로 주소창 변화 대응
- CSS 변수로 모드별 그리드 템플릿 토글:
```css
@media (max-width:1023px) and (orientation:landscape){
  .app{ grid-template-columns: 56px 1fr 38%; grid-template-areas: "nav main log"; }
  .hud{ flex-direction: row; }
}
```

## 6. 개발 로드맵 (단계별)

### MVP (1단계) — "플레이 가능한 최소 루프"
- [ ] 기본 화면/탭 구조 + 반응형 레이아웃
- [ ] 전역 상태 + localStorage 저장
- [ ] 던전 1~3층, 간단한 자동 전투
- [ ] 아이템 드랍 + 인벤토리 + 착용
- [ ] 상점: 수동 진열 + 가격설정 + 즉시 판매
> 목표: 사냥→착용→판매 루프가 한 바퀴 도는 빌드

### 2단계 — "타이쿤 깊이 추가"
- [ ] 손님 방문 시스템 + 시세/수요 변동
- [ ] 방치 수익(오프라인 정산)
- [ ] 상점/인벤토리 확장, 장비 강화
- [ ] 아이템 랜덤 옵션(접사) + 등급별 가치

### 3단계 — "콘텐츠 & 폴리시"
- [ ] 던전 층 확장 + 보스/정예
- [ ] 스킬/소모품, 함정/휴식 이벤트
- [ ] 사운드, 애니메이션(텍스트 연출), 도전과제
- [ ] 밸런싱 및 경제 튜닝

## 7. 밸런스 핵심 포인트
1. **착용 vs 판매 딜레마**가 항상 살아있도록 — 좋은 아이템일수록 둘 다 매력적이게
2. **인벤토리 압박** — 너무 넉넉하면 정리/판매 동기가 사라짐
3. **골드 싱크 확보** — 강화·확장 등 소비처로 인플레 방지
4. **방치 수익 ≤ 능동 플레이 효율** — 직접 플레이가 항상 더 이득이게
5. **층 난이도 곡선** — 강화/장비 없이는 못 넘는 적당한 벽
