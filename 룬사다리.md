# 룬 사다리 (소켓 시스템) — 확장 참고

디아블로식 소켓/룬워드 시스템의 룬 등급표. **전체 1~33위 구현 완료** (효과·드랍률은 [js/data.js](js/data.js) `G.DATA.RUNES`).

> 🖼️ **룬 이미지**: `assets/icon/runs/<rank>.jpg` (1~33 완비).

- 룬은 장비 **소켓**에 박으며, **무기**에 박으면 `wpn` 효과 / **방어구**에 박으면 `arm` 효과 적용(정통 디아블로).
- **큐브**: 같은 룬 3개 → 상위 1개 승급(3:1). 최상위는 승급 불가.
- 소켓에 박은 룬은 **제거 불가(영구)**.
- 데이터 위치: [js/data.js](js/data.js) `G.DATA.RUNES` / 룬워드 `G.DATA.RUNEWORDS`.

> ⚠️ 현재 최상위 이름 **"종언(15위)"** 은 33위 체계에선 중간 등급 → 33위까지 확장 시 이름 재배치(또는 1~15위 테마 정렬) 검토 필요.

## 확장 시 작업 체크리스트
16~33위를 추가할 때:
1. **[js/data.js](js/data.js) `G.DATA.RUNES`** 에 항목 추가 — `{rank, name, iconImg, w, wpn:{}, arm:{}}`. 상위일수록 `w`(드랍가중) 작게, 효과 강하게. (효과는 위 1~15 기조로 보수적 상향)
2. **아이콘**: `assets/icon/runs/` 에 16~33 추가하거나 기존 순환 사용.
3. **큐브 승급**: `G.cube`는 `RUNES.length` 기준 자동 확장 — 별도 수정 불필요(최상위만 승급 불가).
4. **룬워드**: `G.DATA.RUNEWORDS` 에 상위 룬을 쓰는 3·4·5룬 룬워드 추가(엔드게임 빌드정의급).
5. **드랍 깊이 연동(선택)**: 고층일수록 상위 룬 출현율↑ 로직을 `generateRune`에 추가 고려.
6. **밸런스**: 소켓 풀(최대 7부위×5=35) × 상위 룬 → 1000층 기준 인플레 주의. [[combat-effectiveness-rec]] effPower로 검증.
